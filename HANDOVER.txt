================================================================================
1. WHAT THIS PROJECT IS
================================================================================

AgeCX AI is an AI "agent coach" for call-center agents (built for Capita SA).
It sits next to the agent during a live call and:

  - Guides them step by step through the correct call process (the "flow").
  - Answers any policy/process question instantly, grounded in real company
    knowledge documents (Retrieval Augmented Generation = RAG).
  - Also answers behavioural/soft-skill questions (empathy lines, tone,
    de-escalation) like a helpful colleague.
  - Has a supervisor dashboard (/admin) showing sessions, common questions,
    and answers agents flagged as unhelpful.

Goal: fewer "please hold while I check with my team leader" moments. Improves
FCR (first call resolution), NPS (customer satisfaction), and handle time.


================================================================================
2. TECH STACK
================================================================================

Backend  : Python 3.12, FastAPI, SQLAlchemy, Postgres + pgvector
LLM      : Anthropic Claude (model: claude-sonnet-4-6)  -> generates answers
Embeddings: Voyage AI (model: voyage-3)                 -> powers RAG search
Frontend : React 18 + Vite (plain JSX, plain CSS, no UI kit)
3D       : Spline (@splinetool/react-spline) for the landing-page robot
Database : Postgres 16 + pgvector, run locally via Docker Compose

Live deployment (already set up):
  Frontend (Vercel): https://agent-coach-topaz.vercel.app
  Backend  (Render): https://agent-coach.onrender.com
  (Local development does NOT need these; see Section 8 for deploy notes.)

6. QUICK START (Windows / PowerShell)
================================================================================

Open PowerShell. Run these in order. You will need TWO terminals at the end
(one for the backend, one for the frontend).

# 0. Get the code
git clone https://github.com/nyamfusima/agent-coach.git
cd agent-coach

# 1. Start the database (Docker Desktop must be running first!)
docker compose up -d

# 2. Backend setup
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
#   --> NOW EDIT backend\.env and paste in the real ANTHROPIC_API_KEY and
#       VOYAGE_API_KEY (see Section 7). Leave DATABASE_URL as the default.

# 3. Load the knowledge base into the database (needs the Voyage key)
python -m app.services.ingestion

# 4. Run the backend API (leave this terminal running)
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload

# 5. In a SECOND terminal, run the frontend
cd agent-coach\frontend
npm install
npm run dev

# 6. Open the app
#    http://localhost:5173            (the agent app)
#    http://localhost:5173/admin      (the supervisor dashboard)

IMPORTANT: open the app at http://localhost:5173 (use "localhost", not
"127.0.0.1") so it matches the backend CORS settings.

--------------------------------------------------------------------------------
macOS / Linux equivalents for steps 2-3 (everything else is identical):
  cd backend
  python -m venv .venv && source .venv/bin/activate
  pip install -r requirements.txt
  cp .env.example .env
  python -m app.services.ingestion
  uvicorn app.main:app --reload --port 8000

7. ENVIRONMENT VARIABLES (backend/.env)
================================================================================

Create backend/.env (copy from .env.example) and fill in the keys. Example:

  # LLM
  ANTHROPIC_API_KEY=sk-ant-...your key...
  ANTHROPIC_MODEL=claude-sonnet-4-6

  # Embeddings
  VOYAGE_API_KEY=pa-...your key...
  EMBEDDING_MODEL=voyage-3

  # Database (default works with docker compose; do not change for local dev)
  DATABASE_URL=postgresql://agentcoach:agentcoach@localhost:5432/agentcoach

  # App
  ENV=development

  # CORS (optional for local; used in production)
  CORS_ORIGINS=https://agent-coach-topaz.vercel.app
  CORS_ALLOW_VERCEL=true

NOTES:
  - .env is gitignored (secrets never get committed). Each developer makes
    their own. Get the actual key values from the team lead / password manager.
  - The app fails gracefully if keys are missing: chat returns a clear
    "API key not configured" message instead of crashing.

8. HOW THE DATA WORKS (flows + knowledge)
================================================================================

There are two kinds of content per "process" (call type):

  1. FLOW  (data/flows/<process>.json)
     The step-by-step call guide. Drives the left "Call steps" panel.

  2. KNOWLEDGE (data/knowledge/<process>/*.md|.txt|.pdf)
     Source documents. The ingestion script chunks + embeds these into the
     database so the chat assistant can answer questions about them.

Current processes: billing, device_support, essentials, upgrades, billing_query.

These were generated from the original draw.io process maps (the *.drawio.xml
files) using the converter:

  cd backend
  python ../tools/drawio_to_flow.py      # regenerates data/flows + data/knowledge

After changing any knowledge docs, RE-RUN INGESTION so the DB is updated:

  python -m app.services.ingestion                 # all processes
  python -m app.services.ingestion --process billing   # just one process

The ingestion supports .md, .txt, and .pdf files, prints per-file progress,
and tells you the total chunk count at the end.

9. WHAT WAS BUILT / CHANGED (changelog of this engagement)
================================================================================

DATA PIPELINE
  - Wrote tools/drawio_to_flow.py to parse the 4 "AS IS" draw.io process maps
    (Billing, Device Support, Essentials, Upgrades) into app flow JSON + per-page
    knowledge markdown. Handles swimlanes, decisions (YES/NO branches), and
    reconnects the standard call wrap-up steps.
  - Generated data/flows/*.json and data/knowledge/*/*.md from the diagrams.

RAG / CHAT (backend/app/services/rag.py)
  - Two answer modes: POLICY (answers only from the documents, never invents
    rules) and BEHAVIOURAL (empathy/tone/phrasing from general best practice).
  - Confidence score from retrieval similarity; a "low confidence" flag drives a
    UI warning to consider escalating (skipped for behavioural answers).
  - Output is formatted for fast reading (short lines, numbered steps, bullets)
    and is dash-free plain text (no markdown symbols).

BACKEND FEATURES
  - Agent sessions store agent name / id / team and start/end times.
  - Thumbs-down feedback endpoint -> flagged_answers table.
  - Supervisor endpoints (/api/admin/...) for sessions, common questions,
    flagged answers.
  - Graceful API-key handling (HTTP 503 with a clear message if keys missing).
  - Ingestion: PDF support, per-file progress, --process flag, rate-limit-safe
    batching for the Voyage free tier, and a fixed text chunker.
  - Light auto-migrations on startup (adds new columns if an old DB exists).
  - CORS is env-driven: localhost (any port) always allowed; production origins
    from CORS_ORIGINS; optional *.vercel.app previews.

FRONTEND FEATURES
  - Agent login (name / agent id / team, saved in localStorage).
  - Header shows the active call type; loading spinners on all async actions.

BRANDING / UI

10. RUNNING IT AGAIN (day to day, after first setup)
================================================================================

You need 3 things running: the database, the backend, the frontend.

  1. Make sure Docker Desktop is running, then:  docker compose up -d
  2. Backend (in backend/, venv active):
        python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
  3. Frontend (in a 2nd terminal, in frontend/):  npm run dev
  4. Open http://localhost:5173

To reactivate the venv in a new terminal (do NOT recreate it):
  Windows:  .\.venv\Scripts\Activate.ps1
  mac/linux: source .venv/bin/activate

11. TROUBLESHOOTING
================================================================================

"Couldn't reach the API / Failed to fetch" in the UI
  -> The backend isn't running, OR Docker/the database is down, OR you opened
     the app at 127.0.0.1 instead of localhost. Check, in order:
       docker compose ps        (should show agent-coach-db-1 running)
       is uvicorn running on :8000 ?
       open the app at http://localhost:5173

"connection to server at localhost:5432 failed: Connection refused"
  -> The Postgres container is not running. Start Docker Desktop, then
     docker compose up -d

"ModuleNotFoundError: No module named 'app'"
  -> You ran uvicorn from the wrong folder. Run it from inside backend/.

"&& is not a valid statement separator" (PowerShell)
  -> You used a macOS/Linux command. Use the Windows commands in Section 6
     (PowerShell uses ; not &&, and .venv\Scripts\Activate.ps1 not source).

"Port 8000 / 5173 already in use"
  -> Something is still running. Windows, free port 8000:
     Get-NetTCPConnection -LocalPort 8000 -State Listen |
       Select-Object -ExpandProperty OwningProcess | ForEach-Object {Stop-Process -Id $_ -Force}

Chat says "I don't have any process knowledge loaded"
  -> You haven't run ingestion, or it failed. Run:
     python -m app.services.ingestion

Voyage error: "rate limited" or "RateLimitError"
  -> Free tier is 3 requests/min. Wait a minute and retry, or add a payment
     method in the Voyage dashboard (free tokens still apply).

Voyage error: "Provided API key is invalid"
  -> Wrong/expired key. Voyage keys start with "pa-". Update backend/.env and
     re-run. The running backend caches settings at startup, so restart it after
     changing keys.

Anthropic error mentioning 'proxies'
  -> httpx is too new. Make sure httpx==0.27.2 is installed:
     pip install "httpx==0.27.2"
